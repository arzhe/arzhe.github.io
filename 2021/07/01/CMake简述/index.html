CMake简述

Posted on 2021-07-01 Edited on 2021-10-28 In cmake工具教程 Views:

cmake是什么

  cmake是跨平台系统构建器CMake的命令行接口工具。若要使用CMake来构建一个项目，生成一个项目构建系统，可以使用cmake来构建项目、安装项目，或者直接运行相应的构建工具（如make）。

CMake的构建系统（Buildsystems）

  构建系统描述了如何使用构建工具将源代码构建成可执行文件和相关库，这个过程是自动化的。例如，构建系统可以是与命令行make工具一起使用的Makefile，也可以是集成开发环境（IDE）的项目文件。为了避免维护多个这样的构建系统，一个项目可以使用CMake语言编写的文件来抽象地定义它的构建系统。在这些文件中，CMake通过一个称为生成器（generator）的后端在本地为每个用户生成一个首选的构建系统。

  为了生成CMake的构建系统，必须遵循以下几点：

Source Tree

  顶级目录包含项目的源文件，该项目使用CMake语言编写的文件来指定其构建系统，从顶级文件CMakeLists.txt开始一步步构建出其构建系统。这些文件定义了需要构建的目标以及它们的依赖项。

Build Tree

  顶级目录用于存储构建系统文件和构建的输出文件（如可执行文件和库）。CMake编写一个CMakeCache.txt文件来指定该文件所在的目录为Build Tree，并且将诸如构建系统的相关配置项存储于其中。

  为了保持源码树（source tree）的“整洁”，可以使用外部编译，也就是使用Build Tree来作为编译的工作目录，这会将所有的构建输出文件置于该目录中。虽然内部编译也支持，但是为了源码树的“整洁”，不建议这样做。

Generator

  构建器会选择一种构建系统来生成。可以使用cmake --help来查看本地支持的一些构建器：

可以使用-G来指定一个构建器，或者默认使用CMake为当前平台选择的一个构建器。当使用命令行构建工具构建器时，CMake会默认编译工具链所需的环境已经在shell中配置完毕。当使用IDE构建工具构建器时，则不需要提供特定的环境。

生成一个项目的构建系统

  使用下列命令行之一运行CMake来指定源码树和构建树，并生成一个构建系统：

  • cmake [<option>] <path-to-source>

    使用当前工作目录作为构建树，<path-to-source>作为源码树。指定的路径可以是绝对路径，也可以是当前工作目录的相对路径。源码树必须包含一个CMakeLists.txt文件，并且不能包含CMakeCache.txt文件，因为后者会将其所在目录标识为构建树。

  • cmake [<option>] <path-to-existing-build>

    使用<path-to-existion-build>作为构建树，并且从它的CMakeCache.txt文件中加载到源码树的路径，该文件必须已经由之前的CMake运行生成。指定的路径可以是绝对路径，也可以是当前工作目录的相对路径。

  • cmake [<option>] -S <path-to-source> -B <path-to-build>

    使用<path-to-build>作为构建树，<path-to-source>作为源码树。指定的路径可以是绝对路径，也可以是当前工作目录的相对路径。源码树必须包含一个CMakeLists.txt文件。构建树如果不存在，将会被自动创建出来。

构建系统生成后，可以使用相应的本地构建工具来构建项目。比如，在使用了Unix Makefiles构建器后，可以直接使用make命令来完成项目的构建。或者使用cmake --build来调用相应的构建器来完成项目的构建。
